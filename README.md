## 🌌 Core Architecture & Capabilities
<table>
<tr>
<td width="50%" valign="top">
### 🛡️ YouTube JS Challenge Solver
Bypasses complex anti-bot mechanisms instantly. Utilizing a specialized **Node Runtime** environment, it executes obfuscated JavaScript payloads in a secure V8 isolate to extract dynamic decipher keys on the fly.
### 🍪 Cookie Auto Update
Maintains flawless authentication states via the COOKIE_URL remote syncer. If a token expires, the system automatically fetches, parses, and injects fresh Netscape/JSON cookies into the pipeline without dropping a single active request.
### ⚡ Node Runtime Integration
Instead of relying strictly on Python regexes, COOKIE API spins up lightweight Node.js child processes for native DOM emulation, ensuring 100% success rates against client-side script evaluations required by yt-dlp.
</td>
<td width="50%" valign="top">
### 🎬 FFmpeg Merge Engine
Audio and Video streams are pulled asynchronously at maximum bandwidth, then merged in-memory or on-disk via highly optimized FFmpeg subprocesses, guaranteeing pristine 4K/8K High-Quality outputs without audio sync issues.
### 🧠 Advanced Caching System
Implements a layered intelligent cache. Media metadata, thumbnails, and search results are heavily cached (Redis/In-Memory) to achieve sub-10ms response times, bypassing external rate limits and lowering bandwidth overhead.
### 🚀 High-Performance FastAPI
Fully asynchronous from the ground up. Utilizing uvicorn and Python asyncio, the API can handle hundreds of concurrent extractions and FFmpeg merges on standard hardware.
</td>
</tr>

## ⚙️ Environment Variables
Configure your instance using the following .env parameters:
| Variable | Description | Default / Example | Required |
|---|---|---|---|
| API_PORT | Port for FastAPI to listen on | 8000 | ❌ |
| API_ENV | Environment mode (dev, prod) | prod | ❌ |
| COOKIE_URL | Remote URL to fetch fresh cookies | [https://your-server.com/cookies.txt](https://your-server.com/cookies.txt) | ✔️ |
| CACHE_TTL | Time to live for search/metadata (sec) | 3600 | ❌ |
| MAX_CONCURRENT | Limit active FFmpeg merge threads | 5 | ❌ |
| NODE_PATH | Path to Node.js executable | /usr/bin/node | ❌ |
## 🚀 Installation Guide
<details>
<summary><kbd>🐧 Ubuntu / Debian Setup</kbd></summary>
```bash
# 1. Update system packages
sudo apt update && sudo apt upgrade -y

# 2. Install dependencies (Python, Node.js, FFmpeg)
sudo apt install -y python3 python3-pip python3-venv ffmpeg curl
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 3. Clone the repository
git clone https://github.com/MAGMAxRICH/cookie-api.git
cd cookie-api

# 4. Setup Python Virtual Environment
python3 -m venv venv
source venv/bin/activate

# 5. Install Python & Node packages
pip install -r requirements.txt
npm install

# 6. Configure Environment
cp .env.example .env
nano .env # Set your COOKIE_URL

# 7. Run the API
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4

```
</details>
<details>
<summary><kbd>☁️ AWS Deployment (EC2 / ECS)</kbd></summary>
> **Tip:** For AWS, it is highly recommended to use the provided Dockerfile.
> 
```bash
# 1. SSH into your EC2 Instance (Amazon Linux 2023 / Ubuntu)
ssh -i "your-key.pem" ec2-user@your-ec2-ip

# 2. Install Docker
sudo yum install -y docker
sudo service docker start
sudo usermod -aG docker ec2-user

# 3. Clone & Build
git clone https://github.com/MAGMAxRICH/cookie-api.git
cd cookie-api
docker build -t cookie-api:latest .

# 4. Run Detached with Restart Policy
docker run -d \
  --name cookie_backend \
  --restart always \
  -p 80:8000 \
  -e COOKIE_URL="https://your-s3-bucket.s3.amazonaws.com/cookies.txt" \
  cookie-api:latest

```
</details>

## 🛡️ Security & Performance
> **Error Handling**
> The API implements global exception handlers via FastAPI middleware. Instead of failing silently or crashing the uvicorn workers during a bad FFmpeg merge or dead proxy, it intercepts OS-level signals and returns a standardized JSON error trace with HTTP 422 or HTTP 500, keeping the main thread fully unblocked.
> 
> **Performance & Node Execution**
> Executing JS challenges directly in Python is slow and error-prone. We isolate JS execution by spawning a Node.js child process that communicates via stdout/stdin. This guarantees zero memory leaks in the Python parent process while taking full advantage of the V8 engine's JIT compilation.
> 
## ❓ FAQ & Troubleshooting
<details>
<summary><b>My downloads are failing with "Sign in to confirm you're not a bot"</b></summary>


Ensure your <code>COOKIE_URL</code> is correctly pointing to a valid, unexpired Netscape format <code>cookies.txt</code>. The API relies on the Auto Update system to pull this file. Check the logs for <code>[COOKIE_SYNC]</code> events.
</details>
<details>
<summary><b>Audio and Video are out of sync</b></summary>


This occurs if FFmpeg is not installed properly or if system resources (CPU) are choked during the multiplexing phase. Verify your EC2/Ubuntu instance has at least 2 vCPUs.
</details>
<details>
<summary><b>Node runtime challenge solver fails</b></summary>


Make sure Node.js v18+ is installed. If using a custom path, update the <code>NODE_PATH</code> environment variable in your <code>.env</code> file.
</details>
